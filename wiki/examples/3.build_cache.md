# Recompile files only if needed

::: entry

Here project also consists of 2 source files (`utils.c` and `main.c`) and one header (`utils.h`). Buildscript rebuilds both source files in parallel and also caches objects that do not need to be changed.

The core of buildscript is almost the same as before - it contains pointer to [`cbuild_proclist_t`](DOC:cbuild_proclist_t) array `procs`, scratchpad [`cbuild_cmd_run`](DOC:cbuild_cmd_run) `cmd`, uses in identical way [`cbuild_cmd_run`](DOC:cbuild_cmd_run) and then also calls [`cbuild_procs_wait`](DOC:cbuild_procs_wait) for sync. But now each call to [`cbuild_cmd_run`](DOC:cbuild_cmd_run) is wrapped into `if` block that checks if [`cbuild_compare_mtime`](DOC:cbuild_compare_mtime) returns value greater than 0. This just checks if modification time of input file (second argument) is newer than output file (first argument).
